# Project Brief — 孩子行为奖惩 iOS App（RewardingSystem）

> 本文件是项目的**持续执行规范标准**。后续所有设计、开发、决策都必须对齐本文。
> 由用户于项目启动时提供，Claude 以纯 loop engineering 方式自主执行。

## 项目目标
做一个"孩子行为奖惩"iOS App。用积分（奖励分 / 惩罚分）记录并引导孩子日常学习和
生活中的好/坏行为，最终帮助孩子养成规范行为习惯。

- 部署目标：安装到用户的 iPhone 和 iPad（universal app，适配两种屏幕）。
- 第一版只管理**一个孩子**，不做多孩子管理功能（但数据结构上为将来扩展留好余地）。

## 核心需求
1. **角色区分（关键）**
   - 家长(Parent)：完整权限——增删改查奖惩规则、给孩子记分（加分/扣分）、管理奖励兑换商店、查看全部历史与统计。
   - 孩子(Child)：只读受限——只能看最近的分数情况、当前总分、最近行为记录；不能修改规则、不能自己加分。
   - 角色切换需有保护（家长入口用 PIN/FaceID），防止孩子自己切成家长。
2. **奖惩规则管理（家长可 CRUD）**
   - 每条规则含：名称、类别（学习/生活/品德等）、正分或负分值、可选图标。
   - 规则可启用/停用。
3. **记分与流水**
   - 家长一键按规则给孩子记一次分，生成带时间戳的行为流水记录。
   - 支持备注、撤销最近一次误操作。
   - 实时累计总分。
4. **孩子端展示**
   - 当前总分 + 最近 N 条行为记录（好/坏用颜色区分）。
   - 简单可视化进度（趋势/本周获得分），鼓励为主。
5. **奖励兑换（token economy）**
   - 家长维护奖励目录（如"看30分钟电视=20分"）。
   - 孩子可看到能兑换什么、还差多少分；兑换需家长确认。

## 业界成熟实践（直接采纳）
- ABA 代币经济模型：目标行为→得分→兑换奖励 闭环，正向激励为主。
- 轻量游戏化：徽章/连击/里程碑。
- 规则与奖励让孩子"看得见目标"（可视化进度），但奖惩标准只有家长能定。
- UI 对孩子友好：大图标、明亮色彩、少文字；家长端信息密度更高、偏管理。

## 工作方式（Superpowers 全流程）
brainstorm → spec → plan → TDD（先写测试再写实现）→ subagent 执行 →
two-stage code review → systematic debugging → 直到产出可在模拟器跑通的 MVP 才停。

## 纯 LOOP 自主决策规则
1. 不给用户看 spec/plan 等确认——自写、自执行、自迭代。关键决策记录到 DECISIONS.md。
2. 技术选型自主（推荐 Swift + SwiftUI + 本地存储 SwiftData/Core Data，单机离线优先，无后端），理由记 DECISIONS.md。
3. 遇任何选择（数据模型/UI/积分默认值/徽章规则/PIN vs FaceID 等）直接选最优并继续，记 DECISIONS.md。
4. 先用 sample 数据（示例孩子 + 示例奖惩规则 + 示例奖励）跑通全流程，字段格式写进 README。
5. 全程 loop，遇错进入 systematic debugging 自行解决，不抛普通编译/测试报错给用户。
6. Claude 负责把代码写到"Xcode 模拟器能编译运行并通过测试"；真机安装(签名/证书/Apple ID)由用户做，步骤写进 SETUP.md。

## 交付标准
- 可在 iOS 模拟器编译运行的 MVP（universal，适配 iPhone + iPad），核心逻辑有测试覆盖。
- README：架构、数据模型、示例数据说明。
- DECISIONS.md：所有自主决策及理由。
- SETUP.md：用户需做的全部事项（Xcode 打开、配置签名、安装到 iPhone/iPad）。

## 唯一允许停下的时刻
全部做完时，交付 SETUP.md + 运行/安装方式 + DECISIONS.md 摘要。
